# 002.两数相加

## 问题描述
给你两个 **非空** 的链表，表示两个非负的整数。它们每位数字都是按照 **逆序** 的方式存储的，并且每个节点只能存储 **一位** 数字。

请你将两个数相加，并以相同形式返回一个表示和的链表。

你可以假设除了数字 0 之外，这两个数都不会以 0 开头。

**示例 1：**
```
输入：l1 = [2,4,3], l2 = [5,6,4]
输出：[7,0,8]
解释：342 + 465 = 807.
```

**示例 2：**
```
输入：l1 = [0], l2 = [0]
输出：[0]
```

**示例 3：**
```
输入：l1 = [9,9,9,9,9,9,9], l2 = [9,9,9,9]
输出：[8,9,9,9,0,0,0,1]
```

## 解题思路

### 方法：模拟加法运算
由于两个链表是逆序存储的，我们可以直接从链表头部开始遍历，模拟加法运算的过程：

1. 创建一个哑节点作为结果链表的头节点，便于处理边界情况
2. 初始化进位 `jinwei` 为 0
3. 同时遍历两个链表 `l1` 和 `l2`：
   - 计算当前位的和：`num = jinwei + l1.val + l2.val`
   - 更新进位：`jinwei = num / 10`
   - 创建新节点存储当前位的结果：`num % 10`
   - 移动指针到下一个节点
4. 处理链表长度不一致的情况，继续遍历剩余的节点
5. 如果最后还有进位，创建新节点存储进位值
6. 返回哑节点的下一个节点，即结果链表的头节点

**优点：**
- 只需要一次遍历，时间复杂度为O(max(m, n))
- 空间复杂度为O(max(m, n))，主要用于存储结果链表

## 复杂度分析

- **时间复杂度：O(max(m, n))**
  - 其中 m 和 n 分别是两个链表的长度
  - 我们需要遍历两个链表一次，处理每一位数字

- **空间复杂度：O(max(m, n))**
  - 结果链表的长度最多为 max(m, n) + 1（当最后有进位时）
  - 不考虑结果链表的话，空间复杂度为 O(1)

## 注意事项

1. **链表逆序存储的优势**：使得我们可以从链表头部开始处理最低位，符合加法运算的顺序
2. **进位处理**：必须处理好进位，特别是当两个链表都遍历完但还有进位的情况
3. **空指针处理**：当一个链表遍历完而另一个还没有时，需要继续处理剩余链表的节点
4. **哑节点的使用**：简化了结果链表的创建和处理，避免了对空链表的特殊处理
